請記住，您絕不應該將非 GA 的服務和功能用於生產環境。私人和公眾預覽的目的是僅供評估之用。除了缺乏服務等級協議（SLA）和正式支持外，還可能存在其他尚未發現或修復的問題。想想 #ChaosDB 漏洞，這是由 CosmosDB 的一個公眾預覽功能引起的。

以下是您應該注意的一些最重要事項：

| Area           | 私人預覽                 | 公開預覽                                                                   | GA                                 |
| -------------- | ------------------------ | -------------------------------------------------------------------------- | ---------------------------------- |
| SLA            | 無                       | 無                                                                         | 有                                 |
| Support        | 無                       | 受限支援                                                                   | 正式支援                           |
| 對誰可用？     | 僅對少數客戶開放評估使用 | 對所有客戶開放評估使用                                                     | 所有客戶                           |
| 時間線         | 無明確時間表明何時會 GA  | 大部分情況下有 roadmap                                                     | 清楚 roadmap                       |
| 無預期汰除停用 | 會                       | 一般來說不會                                                               | 不會                               |
| 邀請制         | 是                       | 不是                                                                       | 不是                               |
| 如何參與計畫   | 由產品團隊提出邀請       | 使用 [Azure Portal for Public Previews](https://preview.portal.azure.com/) | 直接在 Azure Portal 上使用，無限制 |

# 小技巧

- 每個 Azure 服務和功能在公眾或私人預覽中都有特定的條款和條件。
- 在大多數情況下，私人預覽不提供支持，而公眾預覽提供有限的支持。
- 有時某個服務/功能可能不遵循路線圖，需要額外的工作。在那時候，它可能不會進入 GA，甚至可能被另一個服務/功能取代。

# 如何訪問公眾預覽？

您可以從 Azure Updates 訪問預覽，那裡有一個專門的標籤 IN PREVIEW。或者，您可以使用 Azure 預覽門戶，公眾預覽功能在那裡可用並準備使用。

# 如何禁用預覽功能？

1. 登錄 Azure 門戶。
2. 打開新的面板。
3. 輸入“preview”一詞。
4. 您將看到一個可用預覽功能的列表，並且每個已啟用的功能旁邊都有“enabled”字樣。
5. 選擇“disable”以關閉預覽功能。
